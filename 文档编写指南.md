      <h1 id="文档编写指南">文档编写指南</h1>
<p>本文档介绍如何为佟派产品贡献和创建文档。它包含以下部分：</p>
<ul>
<li><a href="#%E7%BC%96%E5%86%99%E6%96%B0%E5%8A%9F%E8%83%BD%E6%96%87%E6%A1%A3">编写新功能文档</a></li>

<li><a href="#%E7%BC%96%E5%86%99%E7%A4%BA%E4%BE%8B%E7%A8%8B%E5%BA%8F%E6%96%87%E6%A1%A3">编写示例程序文档</a></li>
<li><a href="#%E7%BC%96%E5%86%99%E4%BC%98%E7%A7%80%E6%96%87%E6%A1%A3%E7%9A%84%E6%8C%87%E5%AF%BC%E5%8E%9F%E5%88%99">编写优秀文档的指导原则</a></li>
</ul>
<h2 id="编写新功能文档">编写新功能文档</h2>
<p>当您为佟派产品提供功能时，必须包含一份功能文档，其中详细描述了功能，如何使用功能，如何解决问题等。功能文档应包含高质量的内容，可用作产品文档，市场营销内容和培训材料，只需最少的编辑。有关帮助您编写最佳内容的资源列表，请参见<a href="#%E7%BC%96%E5%86%99%E4%BC%98%E7%A7%80%E6%96%87%E6%A1%A3%E7%9A%84%E6%8C%87%E5%AF%BC%E5%8E%9F%E5%88%99">编写优秀文档的指导原则</a>。</p>
<p>文档应该采用Markdown格式并伴随您的拉取请求。以下是此功能文档的模板以及每个部分应包含的内容。</p>
<h3 id="说明">说明</h3>
<p>介绍您提供的功能的30秒电梯介绍。写一段简短的文字，总结这个特性的目的和功能。从一到两句的功能概述及解决什么问题开始。</p>
<p>您可以在第二段中提供更多描述性信息。本段应回答以下问题：“此功能<em>做什么</em>？”和“<em>为什么</em>我会用它？”如果您的功能具有一系列功能（安装，配置，管理等），您可以在此简要提及它们，稍后在后面的章节中详细介绍。</p>
<h3 id="用法">用法</h3>
<p>解释该功能适用​​的用户故事/场景。</p>
<h3 id="对安装的影响">对安装的影响</h3>
<p>*功能将改变，影响或执行的文件，软件包，服务或操作。<br>
*您的功能自动安装的依赖项。<br>
*警告或其他重要通知。</p>
<h3 id="升级">升级</h3>
<p>如果您的最新版本中断了兼容性或需要特定的升级步骤，请在此处描述所需的步骤。</p>
<h3 id="设置要求">设置要求</h3>
<p>指定所需的环境（例如，已成功测试的JDK，OS和数据库版本）。如果您的功能在设置之前需要额外的功能（另一个功能等），请在此处提及。</p>
<h3 id="配置">配置</h3>
<p>使用本节列出有关此功能配置的特殊注意事项，包括但不限于权限。即使只需要很少的配置，配置部分也是必需的。</p>
<p>明确解释以下内容：<br>
如何启用/禁用该功能将影响存储配置操作的产品UI与基于文件的配置，以及重新启动会如何影响它（哪些配置需要重新启动，哪些不需要）。</p>
<h3 id="例子">例子</h3>
<p>尽可能简明地展示该功能的功能。用户应该能够通过查看示例来了解您的功能如何解决他们的问题。确保您展示的功能显而易见，您的代码/配置简洁明了，并且包含或提及任何相关的API，配置，数据类型和其他元素。</p>
<h3 id="教程培训实验室">教程/培训实验室</h3>
<p>创建一个用户可以用来学习如何使用该功能解决业务问题的教程。指定目标，介绍业务问题/场景，总结如何使用此功能解决问题，然后提供完成教程所需的完整步骤。根据需要包含后端服务等示例文件。<br>
参考<br>
用户需要功能的API，配置和数据类型的完整列表，以及每个参数的参数。</p>
<p>对于API和配置中使用的所有参数，每个元素应包括：</p>
<p>*数据类型（如果适用）。<br>
*参数/元素的功能描述<br>
*有效值，如果数据类型不明显。<br>
*默认值，如果有的话。</p>
<h3 id="api参考">API参考</h3>
<p>提供与该功能相关的API的完整列表。</p>
<h3 id="配置参考">配置参考</h3>
<p>提供与该功能相关的完整配置列表</p>
<h3 id="限制">限制</h3>
<p>描述基础架构兼容性，版本兼容性等。如果存在已知问题，请在此处将它们包含在自己的标题下。</p>
<h3 id="最佳做法">最佳做法</h3>
<p>突出显示使用此功能的最佳做法。例如，如果您可以通过两种不同的方式进行配置，请指定何时选择一种方式。</p>
<h3 id="测试">测试</h3>
<p>描述并演示如何运行测试以确保该功能从用户的角度正确运行。这不是关于功能或单元测试。这更多关于用户验收测试（UAT）。请考虑用户如何确认该功能的应用方式。</p>
<h3 id="故障排除和常见问题">故障排除和常见问题</h3>
<p>解决用户可能经常询问的问题。概述用户在解决方案时遇到的常见问题。解释用户如何通过故障排除解决问题的根源。该功能产生的错误代码列表，带有原因的问题日志以及异常表格在这里会有所帮助。</p>
